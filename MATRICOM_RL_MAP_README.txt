Please start with Creation_of_the_human_mouse_ortholog_map_using_Ensembl.txt, in which I manually obtained mouse orthologs and their corresponding uniprot ids from Ensemble. I then selected a single uniprot id for each gene, prioritizing in this order: 1.) SwissProt ID, 2.) Trembl ID, and collecting the first instance thereof given the priority schedule. This process is followed in the R file, GENERATION_OF_ORTHOLOG_MOUSE_MAPPING.R. I then generated the matricom rl map using the following files as inputs: final_mart_ortholog_table_ENSEMBL_115.tsv (ortholog conversion map), KEGG-hsa04512_LR-RL.csv (KEGG ECM rl annotations), intlist.rda (interactions), CCgenes2 (genes), mlist.rda (proteins). Note that there are no complexes at this stage, and these will need to be added back into the final function for integration into dominoSignal. The function for generating the final map is defined in CREATE_RL_MAP_MATRICOM.R. I ran some checks at the end of the script to see if the interactions were properly annotated. Although inflated by design- (Matricom DB lists a bunch of protein interactions that aren't annotated as having an RL relationship and are not annotated in either OmniPath or KEGG_hsa04512)- the total number of ECM-associated genes seemed reasonable? Also, genes present in the map were overwhelmingly represented within Lisa's dataset, and there is great representation of structural ECM molecules- (see NODE_HUB_DISTRBUTION_HUMAN_RL_MAP_INTERACTIONS.docx).
